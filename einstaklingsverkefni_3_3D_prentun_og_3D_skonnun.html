Project 3 - Additive manufacturing

This project is all about designing and 3D printing something that you can not manufacture using a 3-axis milling machine and scanning an object in 3D.

Part 0 - Test

Before starting we are supposed to to a test to see the capabilities of the printer. The test can be done in 3 person groups and is documented here.

Part 1 - Idea

Firstly, for the 3D printing project we need to know what we're designing, we need an idea. After thinking for a while and cleaning the house and thinking some more, we get the idea to make a broom handle for the broom at home, see image below. You see, the thing is, ever since we got this broom the blue handle at the top has been slowly falling off and last year I taped it to the broomstick but the tape fell off too not long ago so we need this fixed. We need:

  • A way to make the handle stay put without damaging/removing material (this is additive manufacturing)
  • The handle to be strong enough to handle (heh, see what I did there) the load
  • A good grip for the hand
  • To be able to 3D print our solution
  • The solution to be safe (no sharp edges or similar)
  • Bonus: If we can make the handle comfortable

Not thinking about the design requirements, our first idea is to drill a hole through the current handle and the broomstick, put a bolt through there and a nut on the other side, tighten and done. But that won't work, since it's not additive. Our second idea, this time considering the requirements, uses a clamping mechanism, similar to the one you see on hand brakes on bikes, see image below, this clamping mechanism will attach to the broomstick.

After getting this idea approved by the professor, we move on to the next step og the project.

Part 2 - Design

We have our design requirements outlined above and start our design. Making the first sketch there are only 2 things that I'm concerned about and they are how to print this (though I think printing it upside down would be best) and if the clamping will break the plastic when it connects to the upper portion of the device.

On thingiverse, here, there is a solution to the problem of the clamping which is difficult to implement in our case because of a different orientation.

Well, no time to waste so we start modeling. We're using the program Freecad to draw, not fusion360 even though that program is really cool, because this summer I need to use freecad so I might as well practice now. I won't bore you with the details of the drawing in text but here are some screenshots.

Of course to make the dimensions correct we had to measure the actual broomstick.

And then we estimated all the other dimensions such as how far down the shaft of the broom the design needs to go for good grip, how wide my hand is (though I kind of knew that) and such, setting all the dimensions up in Freecad.

When the design was finished it was time to go to manufacturing.

Part 3 - Manufacturing

After finishing the design we bring it into ultimaker cura (the slicer) to set the required settings and export the gcode file to the printer when we realise

We totally forgot to include the height of the 3D printer build volume (which is 120mm) as a design constraint.

This was a quick fix though since we could just alter the distance traveled along the broomstick in the freecad spreadsheet and a moment later we had ourselves a design that fit into the printers build volume.

Originally we set the layer height to 0.175mm, the infill to 20% and add build plate adhesion (since we have a towering print).

The feeling arises that this infill won't be enough since this will be used for a while so we increase it to 40% and the whole thing becomes really dense, we didn't notice at the time but the slicer switched from a grid infill to lines.

Though soon after we realised and switched to the grid again.

Looking at the brim we realise that it's awfully small so just to err on the side of too much stability rather than not enough we increase the number of rounds in the brim to 25.

This results in an estimated print time of 4 hours and 10 minutes using around 40g of filament or 13.31m.

The temperature settings are 205°C for the hot end and 65°C as per the recommendation of the filament producer.

To be able to tighten the slit gap in the design we went and bought a bolt and a nut... Yes we only bought one of each... Yes it was hilarious.

The print time for the shaft was a little longer than the slicer estimated (which is normal for this printer) or around 4 hours and 39 minutes.

Removing the brim the top (this was printed upside down) is a little bit rough around the edges but we can fix that with some sanding.

Part 4 - Assembly

It fits perfectly, looks like you almost don't need the tightening slit.

When putting the bolt into the hole I realise that a 0.2mm tolerance might have been a little too tight because I had to thread the bolt but it's totally fine. Used this website to convert a video to a gif to be able to insert this here below.

The final result is very sturdy and comfortable, seems strong enough, is actually quite comfortable and has a good grip.

Part 4 - 3D scan

It is now time to do the 3D scan which we will do using an app called Qlone

The app requires you to print out a piece of paper with some specific print which we do after receiving it via email.

First we try scanning my glasses just to get a feel for the app. Using the app you need to move your phone around the object on the piece of paper (without crumpling the paper). Meanwhile slowly you build a 3D scan model of the scan.

After scanning the glasses I accidentally deleted the file but that's okay since we were just messing around anyway ^_^ Next up the plan is to scan the 3D printed broomstick handle and compare the scan to the 3D model I made. This is the result I got:

It turns out you need to pay for some premium package to be able to export a 3D model from Qlone so we just export a gif instead. All in all, you can see the scan captures the general shape of the object but does not notice the slit that allows for the tightening around the broomstick shaft. Perhaps using this 3D scan technology could be useful for seeing if something that is a little bit oddly shaped fits into a device you are modeling though I think that with current technology you would probably be better of modeling an approcimate shape in CAD rather than scanning the object. That way you would save time I think.
